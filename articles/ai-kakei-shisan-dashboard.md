## はじめに

家計管理アプリのCSVと証券会社のポートフォリオCSVを毎月手動で確認するのが面倒になってきました。
せっかくならまとめて可視化したい、でも専用サービスには登録したくない、サーバーも立てたくないと考えました。

そこで**Claude Cowork**に全部投げた結果、HTMLファイル1枚のダッシュボードが爆速で完成しました。
しかも途中で発生したエラー対応も、ブラウザ自動化も、「やって」と言ったらやってくれました。

この記事では「**私が何を指示したか**」にフォーカスして書きます。技術的な実装はほぼClaudeが行いました。

---

## できあがったもの

![完成した家計・資産ダッシュボード](/images/ai-kakei-shisan-dashboard/dashboard-screenshot.png)
*※ダッシュボード上の金額・銘柄等のデータはすべてダミーです*

- 家計管理アプリの収支CSVを読み込んで月次収支・カテゴリ別支出をChart.jsで可視化
- 証券会社のポートフォリオCSVを読み込んで銘柄別評価損益を表示
- 配当金データとの統合（利回り計算）
- **家計管理アプリのCSVダウンロードをブラウザ操作で自動化**
- 最新の保有銘柄の株式動向をウェブ検索してまとめて表示

ローカルのHTMLファイル1枚で完結します。サーバーは不要で、データも外部に出にくい構成です。

## 使ったツール

**Claude Cowork**のみ。

ClaudeのデスクトップアプリでCoworkモードを使うと、ローカルのフォルダにアクセスしてファイル操作・コード実行・ブラウザ操作まで一気通貫で実行してくれます。

## 実際にやったこと：指示の流れ

### Step 1: まず雑に全体像を投げる

最初の指示はかなり雑でした。

```
mfフォルダ下で主にタスクをして。
downloadフォルダに収支明細csvがあり、家計管理アプリのがあるので、それを分析。
mfのしたには証券会社のcsvもあるのでそれを一緒にまとめて分析する
```

ファイル名も構造も伝えていません。それでもClaudeはフォルダを自律的に探索し、CSVの中身を確認し、エンコーディング（UTF-8 BOMとCP932が混在していました）まで自動判定してPythonで分析を始めました。

**ポイント：最初から細かく指定しなくて大丈夫です。「何をしたいか」だけ伝えれば探索から始めてくれます。**

---

### Step 2: ファイル管理のルールを追加

```
mfフォルダにCSV管理するようにして。ファイル名を日時をつけて保管して
```

これだけで：
- `mf/家計/` と `mf/証券/` のサブフォルダを作成
- `家計_収支明細_202508.csv` のような命名規則でリネーム・整理

まで自動でやってくれました。

---

### Step 3: ダッシュボードを作る

```
レポートにまとめてください
```

この指示に対して「どんな形式にしますか？」と聞かれたので：

```
HTMLダッシュボード。全期間まとめて
```

と返したところ、複数タブ構成（概要・家計分析・カテゴリ・ポートフォリオ・配当金）のダークテーマHTMLが生成されました。Chart.js 4.4.1をCDNから読み込む構成で、サーバー不要でブラウザで開くだけです。

---

### Step 4: エラーが出たらコピペするだけ

ブラウザで開いたところ、コンソールにエラーが出ました。

```
Uncaught ReferenceError: net_vals is not defined
```

このエラー文をそのままチャットに貼り付けたら、原因特定から修正まで全部やってくれました。私はエラー文をコピペしただけです。

（内部的には、PythonのJSコード生成でf-stringと波括弧のエスケープが衝突していた問題でしたが、そこは知らなくても大丈夫です）

**ポイント：エラーメッセージをそのままコピペするだけで直ります。何が悪いか分からなくても大丈夫です。**

---

### Step 5: データを追加するときも自然な日本語で

毎月データが増えるたびに：

```
downloadに9月から12月のcsvを追加しました。レポートにまとめてください
```
```
2月分もダウンロードしたので月途中だけど追加して
```

こう言うだけで、新しいCSVの読み込み・ダッシュボードの更新・「2月途中*」のような注釈付与まで自動でやってくれました。

---

### Step 6: 分析の質問も日本語で投げる

ダッシュボードを作るだけでなく、そのまま分析相談もできます。

```
一年間の配当金、利回りは？
```
→ 配当金合計・取得価額基準利回り・時価基準利回りを計算して説明してくれました

```
医療費の総額は？
```
→ カテゴリ別に集計し、支出の傾向を指摘してくれました

```
〇〇制度を使う必要ある？確定申告したときとの違いはある？
```
→ 制度の違いを、自分のデータに基づいて比較して説明してくれました

---

### Step 7: ブラウザ操作でCSVダウンロードを自動化

毎月手動でCSVをダウンロードするのが面倒になって聞いてみました。

```
ブラウザでCSVダウンロードを自走できるの？
```

「できます」と返ってきたので、Claude in Chromeという拡張機能をインストールして、ブラウザにログイン済みの状態で：

```
取得してください
```

と言ったところ：
1. 家計管理アプリのページに自動ナビゲート
2. 画面下部のダウンロードボタンをクリック
3. 「ダウンロードしてよいですか？」と確認を求めてくる
4. 「はい」と答えたらCSVをダウンロード
5. そのまま所定のフォルダにリネームして移動

まで自動でやってくれました。

**ポイント：パスワード入力などセンシティブな操作は求めてきません。ログイン済みの状態で使います。**

---

### Step 8: 所有銘柄の最新動向をまとめて調査

```
所有株式銘柄の最近の動向をまとめたページを追加したいな
```

これだけで、保有銘柄すべてについてウェブ検索をかけ、各銘柄の最新トピック・シグナル（↑↑/↑/→/↓/↓↓）・見通しをカード形式でダッシュボードに追加してくれました。

## 指示のコツまとめ

| やったこと | 指示のしかた |
|-----------|-------------|
| 初回分析 | フォルダ名と「分析して」だけ |
| ファイル整理 | 「日時付きのファイル名で管理して」 |
| ダッシュボード作成 | 「HTMLダッシュボードにして」 |
| エラー修正 | エラー文をそのままコピペ |
| データ追加 | 「〇月分追加しました」 |
| 分析相談 | 日本語で普通に質問 |
| ブラウザ自動化 | 「自動化できる？」→「やって」 |
| 市況調査 | 「最近の動向をまとめて」 |

**共通して言えること：「何をしたいか」を自然な日本語で言えば大丈夫です。「どうやって実装するか」を考える必要はありませんでした。**

## 技術スタック（Claudeが選んだ）

指定していないのにClaudeが選択・実装した技術要素です：

- **Python / pandas** — CSVの読み込み・集計（エンコーディング自動判定含む）
- **Chart.js 4.4.1（CDN）** — グラフ描画
- **HTMLシングルファイル** — サーバー不要構成
- **Claude in Chrome** — ブラウザ自動操作によるCSVダウンロード

自分が指定したのは「HTML1枚にして」と「ダークテーマで」くらいでした。

## これからの個人用ツール・レポート作成の形

このプロジェクトを通して感じたのは、AIを使った**「使い捨て（One-off）ツールの究極の個人化」**です。

### 1. 究極の「サーバーレス」とセキュリティ
家計や証券のデータは機密性が高く、SaaSへのアップロードや自前でVercel等にデプロイするのは心理的ハードルがあります。しかし今回の構成は**「ただのローカルHTMLファイル」**です。ブラウザのローカルアクセス（`file://`）だけで動作し、外部通信はCDNからのライブラリ取得のみです。運用次第でリスクを低く抑えやすい、個人用途向けの構成だと感じました。

### 2. 環境構築・保守コストからの解放
ReactなどでSPAを作れば綺麗なダッシュボードは構築できますが、`npm install` や `npm run build` を数ヶ月後に自分のPCで叩き直すのはかなりの苦痛です。今回AIに作らせたのは、**Node.jsすら不要で、テキストエディタで開いてデータをコピペすればすぐ使える**という、一周回って保守性が最も高い「Vanilla JS + 単一HTML」の構成です。

### 3. 「Python ⇆ Vanilla JS」の連携による超パーソナライズ
データ処理の流れも非常に現代的です。
1. Pythonが、文字コードやフォーマットがバラバラな複数のCSVを `pandas` でクレンジング・集計します。
2. 集計結果を、HTMLファイル内の `<script>` タグ（変数定義）に**直接文字列として書き込む（ハードコードする）**形です。

つまり、AIがPythonを使ってデータを焼き付けた静的HTMLビューを毎回自動生成してくれます。「機能を作り込む」のではなく、「自分のためだけの1回きりのレポート画面」をAIにその都度作らせるという、非常に贅沢なコンピューティングの使い方ができるようになりました。

## おまけ：この記事自体と画像作成もAIエージェント（Antigravity）で完結しました

本記事の執筆プロセスと、生々しい個人情報を隠した「Zenn用のダミー画像」の作成自体も、PC上のエージェントAIである **Antigravity** に指示を出して行わせました。

以下は、私がAntigravityに対してチャットで投げた生の指示（プロンプト）の履歴です。

> **私**: 「このプロジェクトでの作業内容をドキュメント化したい。備忘録ですが、あとでZennで公開したい。どのようなツールを使って、どんな指示を出すと作れるかという視点でまとめて。Claude Coworkで作りましたし、ドキュメントはAntigravityで作ったとか。」
> **AI**: （Zenn向けの記事構成とMarkdownのドラフトを自動生成）
> 
> **私**: 「できあがったHTMLをブラウザで表示して、その画面の画像をモザイクで加工できます？」
> **AI**: （ブラウザ操作エージェントが自動でブラウザを開いて画面を撮影し、画像をモザイク加工）
>
> **私**: 「タイトルや『家計アプリ名』『証券会社名』って文字が見えているのでそこもマスクして」
> **AI**: （HTMLを直接編集し、テキストを黒塗りに置換して再撮影）
>
> **私**: 「数値はダミーでランダムにできる？」
> **AI**: （Pythonスクリプトを自動生成して実行し、ファイルの数値を乱数を用いたダミーデータに置換）
>
> **私**: 「データが生々しいな。10倍にして」
> **AI**: （スクリプトを修正して数値を10倍スケールにし、ローカルサーバーを立てて再撮影）
>
> **私**: 「キャプチャの図に数値はダミーですと注意書きしましょう」
> **AI**: （Markdown記事の該当箇所にテキストを自動追記）

このように、**「ブログ公開用のスクリーンショットを撮って画像編集ソフトで個人情報を加工する」という地味で面倒な手作業すら、すべて「チャットで指示するだけ」**でAIエージェントがエディタ、ターミナル、ブラウザをシームレスに操作して完遂してくれました。

## おわりに

「AIに何でもやってもらう」と聞くと大げさに聞こえますが、実際にやってみると**「指示の粒度は雑でいい」**というのが一番の発見でした。

エラーはコピペするだけで直ります。機能追加は「〇〇したい」と言えば追加してくれます。自分がやったのは「何をしたいか」を言い続けることだけでした。

コーディングができなくても、ターミナルを触れなくても、手元のデータを可視化して分析するループを回せる時代になってきたと実感しました。
